## Examples

### Disabled State

Setting `disabled` on `TextField` propagates the state down to all child components through context — no need to set it individually on each one.

<ComponentExample src="svelte/text-field/TextFieldDisabled.svelte">
  <TextFieldDisabled client:load />
</ComponentExample>

### Required Field

Fields marked as `required` automatically render an asterisk indicator inside the `Label`.

<ComponentExample src="svelte/text-field/TextFieldRequired.svelte">
  <TextFieldRequired client:load />
</ComponentExample>

### Validation

When `invalid` is set, the `FieldError` message is shown and the `Description` is automatically hidden to avoid conflicting messages.

<ComponentExample src="svelte/text-field/TextFieldInvalid.svelte">
  <TextFieldInvalid client:load />
</ComponentExample>

### Full Width

Use `fullWidth` to make the field stretch to fill the width of its container.

<ComponentExample src="svelte/text-field/TextFieldFullWidth.svelte">
  <TextFieldFullWidth client:load />
</ComponentExample>

### Text Area

The `TextArea` component can be used within `TextField` for multi-line input.

<ComponentExample src="svelte/text-field/TextFieldTextArea.svelte">
  <TextFieldTextArea client:load />
</ComponentExample>
